# Plan: Add Voice (2026-07-20)

Add audio playback and pronunciation practice to the Time Machine Console.

## Goals

1. Clicking a sentence plays its audio (practice mode and summary page).
2. A practice button next to the sentence opens a recording interface:
   records at most 15 seconds, or until the user clicks a finish button.
3. After recording, the example audio and the user's recording play in
   alternation until the user stops (capped, see UX below).

## Decisions

### Audio source: TTS first, MP3 override later

- Start with **all TTS** via the Web Speech API (`speechSynthesis`,
  `lang: en-US`). No audio assets are required for the feature to work.
- MP3 files are added later, wherever TTS quality is not good enough.
  The player always tries `audio/<key>.mp3` first and falls back to TTS
  automatically when the file is missing — so adding an MP3 requires no
  code change, just dropping the file in place.
- File naming convention (mirrors the data structure):
  - Practice sentences: `audio/<Pronoun>-<time>.mp3`
    (e.g. `audio/I-now.mp3`, `audio/They-then.mp3`)
  - Quiz sentences (full sentence with the answer filled in):
    `audio/quiz-01.mp3` … `audio/quiz-10.mp3`

### Recording

- `MediaRecorder` with the browser's default container/codec
  (WebM/Opus on Chrome, MP4/AAC on Safari). No MIME type is forced;
  the recording is only played back locally in the same browser.
- The recording stays in memory as a Blob object URL. Nothing is uploaded.
- `getUserMedia` is called only when the user clicks the practice button,
  never on page load. Denial shows a friendly message.
- Requires a secure context: serve over HTTPS or `localhost`
  (e.g. `python -m http.server`), not `file://`.

### Alternating playback loop

- State machine driven by the `ended` event of each clip:
  example ends → 400 ms pause → user recording → 400 ms pause → example …
- Capped at **5 repetitions**, then asks "practice again?" instead of
  looping forever.
- A stop button is always visible; switching pronoun, time, or mode
  stops any recording or loop in progress.

### Quiz mode

- No audio before answering (it would give the answer away).
- After the user answers, the full correct sentence is played as
  reinforcement; the next question loads after both the feedback delay
  and the audio have finished.

### UX

- Recording state shows a pulsing red dot and a shrinking 15-second bar.
- Loop state shows which clip is playing and the repetition count (n/5).
- Test on iOS Safari early: audio continuing after the initial user
  gesture (the loop) and MediaRecorder support are the risk areas.

## Architecture

- `voice.js` — shared playback engine (`VoiceEngine`): MP3-first with TTS
  fallback, single-owner playback (starting a clip stops the previous one),
  guaranteed `onEnd` callback. Used by both pages.
- `index.html` — click-to-play on the displayed sentence, listen/practice
  buttons, recording panel, alternating-loop panel, quiz reinforcement.
- `summary.html` — click any example sentence to hear it (derives the MP3
  key from the card's pronoun and NOW/THEN tag; quiz previews carry a
  `data-audio` attribute).
- `audio/` — MP3 files, added incrementally later (README documents the
  naming convention).

## Phases

1. **TTS playback** — click-to-play on both pages via `VoiceEngine`. (done)
2. **Recording + alternating loop** — practice button, 15 s limit,
   finish button, capped loop with stop/again. (done)
3. **Quiz reinforcement audio** — correct sentence after answering. (done)
4. **MP3 assets** — record or synthesize MP3s for sentences where TTS
   quality is lacking; drop them into `audio/` using the naming
   convention. No code change expected. (later)
